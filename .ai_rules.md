# AI 協同開發規範與專案脈絡 (.ai_rules.md)

## 🤖 核心行為守則 (Token 節約模式)
1. **禁止盲目掃描**：除非主動要求，禁止重複執行全局文件掃描或列出完整目錄樹。專案結構請參考下方【專案地圖】。
2. **精準讀取**：修改/分析程式碼時，僅讀取目標檔案及直接相關的 import 檔案，禁止讀取無關模組。
3. **主動記憶**：在本次對話週期內，請保持狀態記憶，避免重複詢問已知的系統架構。

## 📂 文件與規劃路徑規範
1. **落實專案管理**：所有架構規劃、TODO List、技術文件與開發筆記，**絕對禁止**存放在 Antigravity 的系統/快取目錄。
2. **統一存放路徑**：必須統一存放在本專案根目錄的 `docs/ai_planning/` 資料夾下（例如：`docs/ai_planning/feature-x-plan.md`）。
3. **撰寫語言規範**：所有架構規劃、技術文件與說明文件，**一律使用繁體中文（Traditional Chinese）撰寫**。

## 🚀 任務完成定義 (DoD) & 自動推送
當使用者確認某個功能或 Bug 修改完成（例如輸入 "OK"、"完成"、"可以推了"）後，AI 必須**主動且不需提醒**執行以下步驟：
1. 於 Terminal 執行 `git add .`
2. 執行 `git commit -m "feat/fix: [簡短精確的變更說明]"`
3. 自動推送到遠端倉庫：
   - **GitHub**: `git push origin HEAD`
   - **Hugging Face (HF)**: 若偵測到專案包含 `hf` 遠端（或為 Space/Model 專案），須同時執行 `git push hf HEAD`
